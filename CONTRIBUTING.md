# Contributing to Discrete Dating

Thank you for considering contributing to our project! We appreciate your help and want to make the contribution process as easy as possible for you. Here are the guidelines to follow:

## How to Contribute
1. **Fork the repository**: Use the GitHub interface to create a personal copy of the repository.
2. **Clone the repository**: Clone your forked repository to your local machine using `git clone https://github.com/your-username/discrete-dating.git`.
3. **Create a branch**: Create a new branch for your feature or fix. Use descriptive names (e.g. `feature/add-new-feature`, `bugfix/fix-bug`). You can create a new branch using:  
   `git checkout -b your-branch-name`
4. **Make changes**: Implement your feature or fix. Make sure to follow the coding style used in the project.
5. **Commit your changes**: Commit your changes with a clear message describing what you have done. Use:  
   `git commit -m "Your descriptive message"`
6. **Push to your fork**: Push your changes back to your forked repository with:  
   `git push origin your-branch-name`
7. **Open a pull request**: Navigate to the original repository you want to contribute to, and click on the "New pull request" button.

## Code of Conduct
Please note that this project adheres to a code of conduct. By participating, you are expected to uphold this code. For more details, see the `CODE_OF_CONDUCT.md` file.

## Issues
If you find a bug or have a feature request, feel free to open an issue in the repository. Please provide a detailed description of the problem or feature request.

Thank you for your contributions!
